# SAFETY and Redaction Guidelines

This repository is a public, educational portfolio project. To avoid exposing sensitive data or distributing malware, follow these rules:

- Never commit or store real malware samples in a public repository.
- Redact passwords, private keys, and access tokens from tickets and logs. Use [REDACTED] placeholders.
- Replace real API keys with environment variable placeholders in examples.
- If you need to share samples for collaboration, use secure transfer and limited-access artifact repositories.
- Add a note in tickets when redaction has been performed, including the reason and who authorized it.

If you want help redacting additional files or generating sanitized artifacts (e.g., synthetic PCAPs, faux samples), I can create those.
